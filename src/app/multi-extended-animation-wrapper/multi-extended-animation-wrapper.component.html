Multiple Items Another animation driven by component input.

This animation depends on component inputs and their change detection. Animation input parameters for element opacity and animation duration have a continuous state space, but the animation uses only one "named" discrete state data.

The component input has a multi input which accepts an array with item properties controling item ID, color and opacity. Animation duration is controlled through a separate duration input, which applies to all items.

While opacity changes trigger the animation, duration changes do not trigger the animation. A changed duration will only take effect for the next animation.

Settings

Setting changes take effect with the next triggered transition.
