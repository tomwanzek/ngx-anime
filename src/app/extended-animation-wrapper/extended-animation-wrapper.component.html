Single Item An animation driven by component input.

This animation depends on component inputs and their change detection. Animation input parameters for element opacity and animation duration have a continuous state space, but the animation uses only one "named" discrete state data. The component also accepts a color input, which is not animated.

While opacity changes trigger the animation, duration changes do not trigger the animation. A changed duration will only take effect for the next animation.

Setting changes take effect with the next triggered transition.
